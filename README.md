# Tino 福岡三代同堂旅行 App

手機優先、可安裝、可離線查看文字行程的純前端 PWA。沒有帳號、資料庫或付費 API。

## 本機預覽

不能直接雙擊 `index.html` 測試 PWA，請在資料夾內啟動本機伺服器：

```bash
python3 -m http.server 8080
```

再開啟 `http://localhost:8080`。

## 更新行程

所有 Day 1–Day 7 行程都在 `data.js` 的 `window.ITINERARY`。修改站點名稱、時間、說明、座標後重新提交即可，不必改介面程式。

不要自行填入尚未確認的班次、表演時間、訂位結果或票券狀態。

## GitHub Pages 部署

1. 新增 GitHub repository，將本資料夾全部檔案放在 repository 根目錄。
2. Push 到 `main`。
3. Repository → Settings → Pages → Source 選擇 **GitHub Actions**。
4. workflow 完成後，Actions 頁面會顯示公開網址。

所有資源均採相對路徑，因此 repository 名稱不需寫死，也沒有重新整理路由問題。

## 離線與資料保存

- 首次連網開啟後，行程文字與介面會快取在裝置上。
- 地圖圖磚與 Google Maps 導航仍需要網路。
- 完成勾選存在該裝置的 `localStorage`；清除瀏覽器網站資料後會消失。

## V5 行程編排

- 每日時間軸底端可直接按「＋ 新增景點」，以手機 bottom sheet 先填時間、名稱與交通，再按「更多設定」補充資料。
- 每日預設依有效 `HH:MM` 時間穩定排序；可在編輯模式關閉「依時間自動排序」以手動調整順序。
- 相鄰景點會顯示以座標計算的**直線距離**；沒有座標時會明確提示，且不會猜測道路時間。
- 刪除單一景點會顯示六秒的復原列，保留原本景點 ID 與完成紀錄。

開發時可執行以下不需套件安裝的檢查：

```bash
node --check app.js
node tests/itinerary-core.test.js
python3 -m json.tool manifest.json
```


## V6 快速操作與地點搜尋

- 瀏覽行程時即可直接按景點卡上的「編輯」或「刪除」，不必先切換全域編輯模式。
- 「＋ 新增景點」在時間軸捲動時保持可見，適合旅行現場單手操作。
- 景點名稱輸入兩個字後會延遲 450ms 搜尋 Photon / OpenStreetMap，顯示最多五個地點。
- 搜尋會依當日中心座標及旅行城市偏向；選取結果後自動填入名稱、類型、地址說明與經緯度。
- 公開搜尋服務暫時無法使用時仍可手動輸入與儲存，不會鎖住行程。


## V7 天氣、實際支出與搜尋備援

- 每個景點可輸入實際日圓支出；每日摘要與旅行總覽自動加總。
- 旅行設定可填 YYYY-MM-DD 開始日。Open-Meteo 進入可預報範圍後顯示每日高低溫與降雨機率，快取 30 分鐘。
- Photon 搜尋失敗時可按「改用備援搜尋」切換 Nominatim，並支援唐吉軻德／唐吉訶德／Don Quijote／ドン・キホーテ關鍵字。
- 天氣或地點服務離線時不影響手動新增、編輯與實際支出紀錄。


## V8 App 視窗與 Google Maps 出口

- 手機版鎖定在 100dvh App 視窗，頁面本身不再上下漂移；只有中央內容區可以滑動。
- 向下滑動後 Header 自動縮成精簡列，底部導覽持續固定。
- 新增／編輯景點可隨時開啟 Google Maps 搜尋目前關鍵字。此方式不需要 Google API key，但 Google Maps 選取結果不會自動回填。
- 若要 Google 店家自動完成並回填，需另行設定 Google Places API、計費帳戶及受限網域金鑰。
- 天氣在超過 16 天預報範圍時，改為顯示該日可開始查詢預報的日期。


## V9 大眾化操作

- 新增旅行改為三步驟精靈：旅行名稱與城市、日期、同行成員與住宿。日期會自動建立 1～31 天的空白行程。
- 景點進階設定可貼上完整 Google Maps 網址，免費解析網址中的店名、座標或 query 並回填。Google Maps 短網址受跨網站限制，會提示改貼完整網址。
- Header 顯示已儲存、離線使用中、新版可更新或離線設定失敗等狀態。
- 景點新增第一層維持時間、名稱、交通與實際花費；分享連結、座標和照護提醒收在更多設定。


## V10 Journey-first 介面減法

- 總覽只保留完整旅行封面，不在下方重複成員、住宿、進度與支出卡片。
- 點選任一 Day 會直接進入行程頁並回到頂端。
- 行程頁採固定精簡 Header，不再依捲動臨界點放大／縮小，避免畫面跳動。
- 行程模式隱藏人數、住宿、城市、旅行切換與全域編輯按鈕。
- 每日摘要只保留日期、標題、天氣、站數及實際花費。
- 新增景點改為底部導覽上方的小型浮動按鈕，避免遮住時間軸。


## V10.1 設定頁減法

- 完整旅行封面只出現在總覽。
- 設定頁完全隱藏 Hero，開啟後直接從設定表單開始。
- 移除設定頁重複的 TRIP SETTINGS 小標，縮短頁首佔用。

## V10.4 旅程記憶總覽

- 每趟旅行可選擇一張最重要的照片；照片會在裝置端縮放並壓縮後保存，不上傳外部服務。
- 總覽依日期顯示出發倒數、旅途中 Day 或旅程回顧，並保留精簡的天數、景點數與總花費。
- 多趟旅行會以橫向卡片呈現，點選卡片即可切換旅程。
- 複製旅行只複製行程模板，不複製原旅行的記憶照片。

## V10.5 備份安全與瀏覽狀態

- 離開總覽時自動結束全域編輯模式，行程、地圖與設定永遠先以乾淨瀏覽狀態開啟。
- 總覽只在尚未完整備份或超過 14 天未備份時顯示精簡提醒。
- 設定頁顯示最後完整備份日期，完整備份會包含所有旅行資料與壓縮後的照片。
